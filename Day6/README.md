# Day 6 — Databases

Notes for Day 6 of the [10-Day GCP Learning Plan](../GCP-10-Day-Learning-Plan.md).

> **Big-picture analogy:** If raw storage (Day 5) is a **filing cabinet** where you
> stuff whole files, a **database** is a **smart librarian** who keeps records neatly
> organized and can instantly answer "give me all orders over $100 from June." GCP
> offers different librarians for different jobs — some great at strict ledgers (SQL),
> some at flexible sticky-notes (NoSQL), some at planet-scale.

## Topics
1. [Database Fundamentals](01-database-fundamentals.md) — SQL vs NoSQL, managed vs self-managed, OLTP vs OLAP.
2. [Cloud SQL](02-cloud-sql.md) — managed MySQL / PostgreSQL / SQL Server.
3. [Firestore & NoSQL](03-firestore-and-nosql.md) — flexible document database for apps.
4. [Bigtable & Spanner](04-bigtable-and-spanner.md) — huge-scale NoSQL & global SQL.
5. [Memorystore & Choosing a Database](05-memorystore-and-choosing.md) — caching + a decision guide.

## Day 6 Goals
- [ ] Explain SQL vs NoSQL and managed vs self-managed.
- [ ] Know when to use Cloud SQL, Firestore, Bigtable, Spanner, Memorystore.
- [ ] Understand OLTP vs OLAP (and where BigQuery fits — Day 9).
- [ ] Spin up a Cloud SQL instance and connect from Cloud Shell.
- [ ] Create a Firestore collection and add documents.
